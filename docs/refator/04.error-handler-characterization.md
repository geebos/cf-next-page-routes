# 04.error-handler-characterization

## 发现的问题

`createErrorHandler` 无测试；BusinessError / 500 契约是全站 API 错误形状 SSOT。

## 问题证据和影响范围

| 项 | 说明 |
| ---- | ------ |
| 证据 | 前端 `ApiError` 依赖 `{ code, message, data }`；零覆盖 |
| 影响 | 新增 `src/worker/middleware/error.test.ts` |
| 生产 | 零改动 |

## 最小修改方案

Hono mini app + `app.onError(createErrorHandler())`：

- BusinessError 404/400 透传
- unknown Error → 500「服务器错误」且不泄漏 message
