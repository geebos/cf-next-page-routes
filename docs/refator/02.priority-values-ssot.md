# 02.priority-values-ssot

## 发现的问题

`todo-form.tsx` 本地 `PRIORITY_VALUES = ["low","medium","high"]` 与 `priorityEnum = z.enum([...])` 双源，增删 priority 时 Select 与 Zod 可能漂移。

## 问题证据和影响范围

| 项 | 说明 |
| ---- | ------ |
| 证据 | form L42 与 schema L53 字面重复 |
| 影响 | `src/shared/schemas/todo.ts`、`src/components/pages/todo/forms/todo-form.tsx` |
| 行为 | 不变 |

## 原始设计

form 自维护常量；schema 另写 enum。

## 最小修改方案

```ts
export const PRIORITY_VALUES = ["low", "medium", "high"] as const;
export const priorityEnum = z.enum(PRIORITY_VALUES);
```

form 从 `@/shared/schemas` 导入，删除本地常量。

## Design Challenger 结论

- **verdict:** APPROVE
- 建议用 `as const` 数组再 `z.enum`，优于 `priorityEnum.options` 类型

## TDD 过程

纯常量搬家；无行为分支。typecheck/eslint 验证。

## 最终修改

- schema 导出 `PRIORITY_VALUES`
- form 改为 import

## Code Reviewer 结论

- **APPROVED**
