# 04 — Worker Robustness

**Priority:** Medium–High  
**Source:** [project-review.md](./project-review.md) findings #5, #11; gaps G3, G4, G5  
**Category:** Security (info leak) + Correctness (sqlite / Node entry)

---

## Problem

| ID | Issue |
|----|--------|
| #5 | `createErrorHandler` returns raw `err.message` on 500 → config/SQL leak to clients |
| #11 | `getSqliteDb` TOCTOU: concurrent first requests double-open + double-migrate |
| G3 | `serve.ts` defaults to D1 without binding when `DB_ENGINE` unset → all API 500 |
| G4 | `migrationsFolder: "./drizzle"` is CWD-relative → fails off repo root |
| G5 | No `busy_timeout` / WAL → concurrent writes can surface `SQLITE_BUSY` |

---

## Goals

- Clients never see internal exception text on 500.
- SQLite engine initializes exactly once under concurrency.
- Node entry is hard to misconfigure; migrations resolve reliably.
- Concurrent sqlite writes wait instead of hard-failing when possible.

## Implementation steps

### 1. Generic 500 responses (required)

```ts
console.error(`[InternalError] msg="${err.message}"`, err);
return c.json(
  { code: 500, message: "服务器错误", data: null },
  500,
);
```

Keep full detail only in logs. `BusinessError` path unchanged.

### 2. SQLite init singleton (required)

```ts
let sqliteDb: ... | null = null;
let sqliteInit: Promise<...> | null = null;

export async function getSqliteDb() {
  if (sqliteDb) return sqliteDb;
  if (!sqliteInit) {
    sqliteInit = (async () => {
      // open + migrate + assign sqliteDb
      return sqliteDb!;
    })();
  }
  return sqliteInit;
}
```

On failure, clear `sqliteInit` so a later request can retry (optional but recommended).

### 3. `serve.ts` fail-fast (required)

Options (pick one):

- Default `DB_ENGINE=sqlite` when running under Node serve entry; require `DB_SQLITE_DIR`.
- Or: if engine is `d1` and `c.env.DB` missing, throw a clear startup error before listening.

Do not silently serve with undefined D1 binding.

### 4. Migrations path (required)

Resolve relative to a known root, e.g.:

- `path.join(process.cwd(), "drizzle")` only after documenting “must start from repo root”, **or**
- `path.join(fileURLToPath(new URL("../../../drizzle", import.meta.url)))` / env `DB_MIGRATIONS_DIR`.

Prefer path that works when cwd ≠ repo root when feasible under bundler constraints.

### 5. SQLite pragmas (recommended)

After open:

```ts
conn.pragma("journal_mode = WAL");
conn.pragma("busy_timeout = 5000");
```

---

## Acceptance criteria

- [ ] Forced internal throw → client body `message` is generic `"服务器错误"` (or project equivalent), not stack/config text.
- [ ] Parallel first requests under sqlite: single migrate, stable db.
- [ ] Misconfigured Node start fails with actionable log (missing `DB_SQLITE_DIR` / wrong engine), not opaque per-request 500.
- [ ] Migrations apply when started from documented cwd (or absolute path).
- [ ] Rapid concurrent toggles under sqlite rarely/no longer return raw `SQLITE_BUSY` (timeout waits).
