### R03-001 — Tauri 2 自定义 command 的 ACL 写法不完整

| 字段 | 内容 |
|------|------|
| 问题编号 | R03-001 |
| 提出版本 | v0 |
| 问题描述 | 文档 §2 写「Allow custom commands…（`proxy`；drop or allow `greet` intentionally）」并暗示在 `default.json` 的 permissions 里直接放 command 名，但未给出 Tauri 2 真实 ACL 路径：应用自定义 command **默认并不**像插件那样自动生成 `allow-*` 权限；需在 `src-tauri/permissions/*.toml` 中声明 `[[permission]]`（`commands.allow = ["proxy"]` 等），再把 permission identifier（如 `allow-proxy`）写入 capability。G2 的「not allowlisted… invoke may fail」在「未声明 permission + capability 未引用」时才成立；文档缺这一层会导致实现方误以为写 `"proxy"` 字符串即可。 |

### R03-002 — URL allowlist 未覆盖 DNS rebinding / 解析时与连接时主机差异

| 字段 | 内容 |
|------|------|
| 问题编号 | R03-002 |
| 提出版本 | v0 |
| 问题描述 | §1 要求「Parse URL；allow only configured prefixes；reject link-local / metadata」。仅做 **字符串前缀** 校验无法挡住：① DNS rebinding（allow 的 hostname 先解析到公网再切到 `169.254.169.254` / 内网）；② 字面量 IP 绕过 hostname 前缀；③ IDN / 大小写 / 默认端口 / 尾斜杠规范化差异；④ `http://user@allowed-host@evil` 类 URL 解析陷阱。验收只写「`http://169.254.169.254/` returns error」偏弱，未定义「允许 hostname 但解析到 metadata IP」的预期。 |
| Review Agent 建议 | 明确校验层：至少 normalize（scheme、host lower-case、拒绝 credentials、拒绝非 http(s)），allowlist 以 **host（+ 可选 path prefix）** 为准而非任意 URL 前缀；对 dev 的 `127.0.0.1`/`localhost` 单独 `cfg!(debug_assertions)`。可选进阶：resolve 后拒绝私网/link-local/metadata IP（需说明与企业代理/内网 API 的冲突）。验收补充 rebinding / 字面量 IP / 带 userinfo 的 URL 用例。 |
| Answer Agent 回复 | **采纳 v1 最小 hardening；post-resolve rebinding 进 Out of scope。** 校验顺序写死：(1) `Url::parse` 失败 → reject；(2) scheme ∈ `{http, https}` 以外 → reject；(3) **有 userinfo（username/password）→ reject**；(4) host 取 `host_str()` lower-case；缺 host → reject；(5) 匹配 allowlist：**exact host** 或（仅 debug）`localhost` / `127.0.0.1` / `::1`，可选 path prefix 仅用于生产 API host；(6) **字面量 IP**：release 仅当 IP 等于已配置的 API host 字面量时放行；**metadata / link-local 字面量一律拒绝**（`169.254.0.0/16`、`metadata.google.internal` 等）；debug 允许 `127.0.0.1`/`::1`。(7) **不做**连接前 DNS resolve 后二次校验（企业/内网 API、复杂度）——写入 Out of scope；不宣称挡住经典 DNS rebinding。Allowlist **不是**任意 URL 字符串前缀。 |
| 文档调整说明 | §1 重写为 normalize + host allowlist 步骤；AC 补 userinfo / 字面量 metadata IP / 非 allow host；rebinding post-resolve 进 Out of scope。 |
| 当前状态 | 已关闭 |
| 关闭版本 | v1 |

### R03-003 — 未处理 reqwest 默认跟随重定向导致的 allowlist 绕过

|------|------|
| 问题编号 | R03-003 |

| 文档调整说明 | §1/§5 与 Client builder 示例强制 `Policy::none()`；AC 增加 302 不跟随用例。 |

### R03-005 — Set-Cookie 多值 IPC 方案未落到可实现的 wire format

| 提出版本 | v0 |

## Decision（产品与安全，v1 锁定）

| 议题 | 决议 |
|------|------|
| Test 页定位 | **dev/debug 工具**，不是开放代理 |
| SSRF 策略 | `proxy` **仅** allow：配置的 API host + debug 下 localhost/`127.0.0.1`/`::1` |

| 业务 API | **只** `apiFetch` → plugin-http → `http:default` |
| 重定向 | `redirect(Policy::none())`，3xx 原样返回 |
| Set-Cookie IPC | `headers: Vec<(String,String)>` / JS `[string, string][]` + `Headers.append` |
| Timeout | **30s** total / **10s** connect；`OnceLock` 共享 Client |
| App command ACL | 默认可用，但 v1 **AppManifest 仅 expose `proxy`**（drop `greet`）；可选 `allow-proxy` permission 写入 capability |

## Architecture（双通道）

| 通道 | 调用方 | 前端入口 | Rust / 插件 | ACL | URL 策略 | Timeout | Headers / body |
|------|--------|----------|-------------|-----|----------|---------|----------------|
| **业务** | Todo、warmup health | `apiFetch` → `adapter/fetch` | `tauri-plugin-http` | `http:default` allow URLs | 仅 capability 列出的 URL pattern（默认 `https://template.geebosblog.com/*` + 与 base URL 对齐的 host） | 插件默认；本篇不改 | 插件语义；**本篇不改** Set-Cookie/null-body |
| **Test / debug** | `/test` 页 | `request()` → `invoke("proxy")` | 自定义 `proxy` + reqwest | AppManifest / optional `allow-proxy`；**独立于** `http:default` | host allowlist（API host + debug loopback） | **10s connect / 30s total** | Vec 对 headers；null-body 204/205/304 |

业务 **永不** 调用 `proxy`。未来若业务改走 proxy，另开文档，禁止 silent scope creep。

## Problem

The Tauri stack has an open HTTP path and incomplete capability / adapter hardening:

| ID | Issue |
|----|--------|
| #4 | Rust `proxy` accepts any URL — intentional bypass of http plugin allowlist (SSRF) |
| G2 | App commands are allowed by default in Tauri 2, but project has no explicit lockdown; plugins still need `http:default`. v1 tightens app manifest + documents ACL correctly. |
| G1 | `http:default` only allows `https://template.geebosblog.com/*` — must stay aligned with `DEFAULT_TAURI_BASE_URL` / `NEXT_PUBLIC_BASE_URL` for **plugin-http** |
| #7 | JS `request()` only nulls body for 204/304 — 205 throws in `Response` constructor |
| #8 | Multi `Set-Cookie` folded with `", "` — broken cookies |
| G8 | New `reqwest::Client` per call, no timeout — hung peers freeze UI |
| Redirect | Default reqwest follows redirects → allowlist bypass via `Location` |

---

## Goals

- No unrestricted SSRF from the webview (`proxy` host allowlist + no auto-redirect).
- Explicit app-command surface (`proxy` only) and `http:default` hosts aligned with business base URL.
- Proxy response reconstruction: null-body statuses + multi-value headers without Set-Cookie fold.
- Proxy requests fail within fixed timeouts instead of hanging forever.

## Implementation steps

### 1. Restrict `proxy` URLs (required) — channel: proxy

Before `client.request`:

1. `Url::parse(&args.url)` — fail → `Err("proxy error: invalid url")`.
2. Scheme must be `http` or `https`.
3. Reject if URL has username or password (userinfo).
4. `host = host_str().to_ascii_lowercase()`; missing host → reject.
5. Allow if host matches **configured API host(s)** (compile-time / const list aligned with production `DEFAULT_TAURI_BASE_URL` host, e.g. `template.geebosblog.com`). Optional path-prefix check only if product needs it; v1 host match is enough.
6. **Debug only** (`cfg!(debug_assertions)`): also allow `localhost`, `127.0.0.1`, `::1` (any port).
7. Reject literal link-local / metadata targets (e.g. host `169.254.169.254`, `metadata.google.internal`) even if somehow listed.
8. **Not** arbitrary string prefix matching on full URL.

Update command doc-comment: independent of plugin-http allowlist; enforced by this host allowlist. Test page is **not** a general-purpose open proxy.

### 2. Capabilities + app command ACL (required)

**App commands (Tauri 2 fact + v1 policy):**

- Fact: commands registered via `generate_handler!` are **allowed by default** for all windows.
- v1 policy: lock the app command surface in `build.rs`:

```rust
// build.rs
fn main() {
    tauri_build::try_build(
        tauri_build::Attributes::new()
            .app_manifest(tauri_build::AppManifest::new().commands(&["proxy"])),
    )
    .expect("failed to run tauri-build");
}
```

- Drop `greet` from `generate_handler!` and from the product (sample-only).
- Optional readability layer — `src-tauri/permissions/proxy.toml`:

```toml
[[permission]]
identifier = "allow-proxy"
description = "Allows the debug HTTP proxy command used by /test."
commands.allow = ["proxy"]
```

- `capabilities/default.json` may list `"allow-proxy"`; **must not** put bare `"proxy"` as if it were a plugin permission id.
- **Plugin HTTP (G1 / business):** keep expanding `http:default` `allow` to every host `apiFetch` hits:

```json
{
  "identifier": "http:default",
  "allow": [
    { "url": "https://template.geebosblog.com/*" }
  ]
}
```

If desktop dev uses plugin-http against a local worker, add that host under **debug** builds only (document in README); do not open `*` .

App command ACL and `http:default` are **two systems** — neither substitutes for the other.

### 3. Redirect policy (required) — channel: proxy

Shared client:

```rust
reqwest::Client::builder()
    .connect_timeout(Duration::from_secs(10))
    .timeout(Duration::from_secs(30))
    .redirect(reqwest::redirect::Policy::none())
    .build()
```

Return 3xx status + headers (including `Location`) to the frontend; do **not** follow.

### 4. Null-body statuses in `request.ts` (required) — channel: proxy

```ts
// Required: 204, 205, 304. 101/103 optional (safe to include).
const NULL_BODY = new Set([204, 205, 304]);
// optional: also 101, 103

const body = NULL_BODY.has(res.status)
  ? null
  : new Uint8Array(res.body);

const headers = new Headers();
for (const [k, v] of res.headers) {
  headers.append(k, v);
}

return new Response(body, { status: res.status, headers });
```

- Status **205** with upstream body → still pass `null` (discard), never throw.
- Proxy failures stay `invoke` rejection / thrown Error — **never** encode as HTTP status `0` in `ProxyResponse`.

### 5. Set-Cookie / multi-value headers (required) — channel: proxy

Wire format (selected):

```rust
// Rust
struct ProxyResponse {
    status: u16,
    headers: Vec<(String, String)>, // JSON: [string, string][]
    body: Vec<u8>,
}
```

- Iterate `resp.headers()` in order; `push((name, value))` for each; **never** fold with `", "`.
- JS: `Headers.append` per pair (see §4).
- **No cookie jar** in proxy (session bookmark = [01](./01.d1-session-bookmark.md), business path = plugin-http).

### 6. Timeouts + shared client (required) — channel: proxy

| Constant | Value | Applies to |
|----------|-------|------------|
| `PROXY_CONNECT_TIMEOUT` | **10s** | TCP/TLS connect |
| `PROXY_TIMEOUT` | **30s** | Full request including body download |

- One `Client` in `OnceLock` (or `LazyLock`); built once with timeouts + `Policy::none()`.
- Errors: `proxy timeout: …` when timeout; else `proxy error: …`.
- Frontend: existing `test.tsx` `finally` clears loading; timeout must surface as toast via `catch` (no infinite spinner).

### 7. Optional efficiency

- Avoid `Array.from` entire body if a binary IPC path exists; not blocking for this security batch.

## Acceptance criteria

- [ ] `invoke("proxy")` to `http://169.254.169.254/` → error (not success body).
- [ ] `invoke("proxy")` with userinfo URL (`http://user@host/...` / credential form) → error.
- [ ] Non-allowlisted host → error; allowlisted production host succeeds from Test page.
- [ ] Debug build: `http://127.0.0.1:…` / `localhost` allowed; release build: loopback **not** allowed unless host is configured as API host.
- [ ] Allowlisted host responds `302` + `Location` to non-allow target → client **does not** request Location target; frontend sees 3xx.
- [ ] AppManifest: only `proxy` exposed; `greet` removed or denied; `/test` works for allowlisted URL.
- [ ] Todo `apiFetch` host is on `http:default` allowlist (aligned with `DEFAULT_TAURI_BASE_URL`).
- [ ] Status 204 / 205 / 304 reconstruct without throwing; 205 with body discards body.
- [ ] Multi `Set-Cookie` not comma-joined; each cookie is a separate header pair.
- [ ] Blackhole / non-responding host fails within **≤ 30s + small ε**; Test UI shows toast, not infinite spinner; error string starts with `proxy timeout` when timed out.
- [ ] Proxy hard errors remain thrown/`invoke` reject — not `ProxyResponse.status === 0`.
