# 01 — D1 Session Bookmark & Cookies

**Priority:** High  
**Source:** [project-review.md](./project-review.md) findings #1, #6, #9; gap G1  
**Category:** Correctness (session continuity)

---

## Problem

D1 read-your-writes depends on the `d1_bm` bookmark cookie. Three related defects break that path:

1. **Empty string overwrite** — `session.getBookmark() ?? ""` writes `""` when bookmark is null. Next request uses `getCookie(...) ?? DEFAULT`, but `""` is not nullish, so `withSession("")` runs instead of `"first-unconstrained"`. Health/warmup can wipe a good bookmark.
2. **Always `secure: true`** — browsers refuse Secure cookies on `http://localhost`, so local wrangler never persists the cookie.
3. **Tauri / cross-origin** — `apiFetch` does not set credentials; Tauri absolutizes API URLs. Cookie may never be stored or sent; http allowlist may also block non-default hosts (see [03](./03.tauri-proxy-security.md)).

---

## Goals

- Empty or missing bookmark always falls back to `first-unconstrained`.
- Cookie is set with `secure` only when the request is HTTPS (or production policy).
- Document (or implement) how Tauri clients participate in bookmark sessions.

## Implementation steps

### 1. Coalesce empty bookmark (required)

On **read**:

```ts
const raw = getCookie(c, BOOKMARK_COOKIE);
const bookmark = raw && raw.length > 0 ? raw : DEFAULT_BOOKMARK;
// or: (getCookie(c, BOOKMARK_COOKIE) || DEFAULT_BOOKMARK)
```

On **write**:

```ts
const next = session.getBookmark();
if (next) {
  setCookie(c, BOOKMARK_COOKIE, next, { ...options });
} else {
  // omit Set-Cookie, or deleteCookie / set to DEFAULT explicitly
  // Prefer: do not write empty string
}
```

Do **not** use `?? ""` as the cookie value.

### 2. Conditional `secure` (required)

```ts
const isHttps =
  c.req.header("x-forwarded-proto") === "https" ||
  new URL(c.req.url).protocol === "https:";

setCookie(c, BOOKMARK_COOKIE, value, {
  httpOnly: true,
  secure: isHttps, // or: process.env.NODE_ENV === "production" && isHttps
  sameSite: "lax",
  path: "/",
  maxAge: 60 * 60,
});
```

Local HTTP must be able to store `d1_bm`.

### 3. Tauri cookie strategy (decide + document)

Pick one and document in README / this file:

| Option | When |
|--------|------|
| A. Desktop uses sqlite local worker | Bookmarks irrelevant; no change to apiFetch credentials |
| B. Desktop talks to remote D1 API | Ensure plugin-http / credentials can jar cookies for API host; align allowlist (G1) |
| C. Explicitly no RYW in Tauri | Document that create-then-list may lag; accept |

If B: set credentials appropriately for the adapter and verify with create → immediate list.

## Acceptance criteria

- [ ] After `GET /api/health`, a previously valid `d1_bm` is not replaced with empty string.
- [ ] `getCookie` empty / missing → `withSession("first-unconstrained")`.
- [ ] Local `http://localhost:8787` (or pages origin) can store and resend `d1_bm`.
- [ ] Production HTTPS still sets `secure: true`.
- [ ] Tauri path has an explicit decision (A/B/C) written down; if B, create-then-list RYW verified.
