## Implementation steps

### 1. Trailing slash policy (required)

**Stack facts (this repo):**

| Layer | Current |
|-------|---------|
| `package.json` | `wrangler` `^4.103.0` |
| `next.config.ts` | `output: "export"`, `trailingSlash: true` → pages emit `…/todo/index.html` |
| Path helpers | `localizePath` always ends with `/`; `Seo.createLocalizedUrl` always ends with `/` |
| `wrangler.jsonc` assets | `html_handling: "drop-trailing-slash"` (**misaligned**), `run_worker_first: ["/api/*"]` |

**`html_handling` legal enum** (`node_modules/wrangler/config-schema.json` for wrangler ^4.103):

| Value | Behavior (folder index present) | Use in this repo? |
|-------|----------------------------------|-------------------|
| `auto-trailing-slash` | Serve/redirect toward “natural” form (file no slash, folder with slash); redirects are **307** | No — not an explicit product policy; easy to misread vs helpers |
| `force-trailing-slash` | `/en/todo` → **307** → `/en/todo/`; `/en/todo/` → **200** | **Yes — only allowed value** with `trailingSlash: true` |
| `drop-trailing-slash` | `/en/todo/` → **307** → `/en/todo`; `/en/todo` → **200** | No — fights Next + `localizePath` + Seo |
| `none` | No HTML trailing-slash normalization | No — export URLs rely on index handling |

**Decision:** set only:

```jsonc
"assets": {
  // ...
  "html_handling": "force-trailing-slash"
}
```

Do **not** leave “or another mode…” options. Status codes on CF assets normalization are **307** (not 308) per [Cloudflare HTML handling](https://developers.cloudflare.com/workers/static-assets/routing/advanced/html-handling/).

**Assets vs worker boundary:**

- Static HTML under `assets.directory` (`out`) is handled by the assets layer (`html_handling`, `not_found_handling`).
- `run_worker_first: ["/api/*"]` only forces `/api/*` into the Worker first. Page routes are **not** rewritten by worker-first; their trailing-slash behavior is purely assets `html_handling`.

**Cookie / session side effects:**

| Concern | Status |
|---------|--------|
| `d1_bm` (D1 bookmark) | `path: "/"` in `src/worker/middleware/db.ts` — **unaffected** by HTML trailing-slash redirects |
| Locale preference | `localStorage`, not a cookie |
| Path-scoped page cookies | Avoid setting cookies whose `Path` differs only by trailing slash on page routes; prefer `path: "/"` or API-scoped paths under `/api` |
| HTML 307 vs API | Browser follows 307 on HTML; API cookies are set on API responses under `/api/*` and are separate from assets redirects |

### 2. Todo page Section (required)

In `src/pages/[locale]/todo.tsx`:

- Import `Section` from `@/components/ui/section` (same as test page).
- Replace create-form and list wrappers:

```tsx
<Section className="rounded-lg border border-border bg-card p-4">
  <CreateTodoForm ... />
</Section>

<Section>
  <List ... />
</Section>
```

**Class rules:**

- Root always includes `flex flex-col gap-2` from `Section` (`cn("flex flex-col gap-2", className)`).
- That extra layout is **intentional** — align todo with test page; do **not** treat it as a regression against pre-migration native `<section>`.
- Keep create’s visual chrome via `className` (`rounded-lg border border-border bg-card p-4`).
- `title` / `action` are optional; todo does **not** pass them.

## Acceptance criteria

- [ ] `wrangler.jsonc` has `assets.html_handling: "force-trailing-slash"` (no other enum value).
- [ ] After `pnpm build` + wrangler/assets preview (or deploy), trailing-slash checks with **headers**:

  ```bash
  # no trailing slash → 307 to canonical form
  curl -sI "https://<host>/en/todo" | head -n 20
  # expect: HTTP/… 307
  # expect: Location: …/en/todo/   (or absolute URL ending in /en/todo/)

  # with trailing slash → 200
  curl -sI "https://<host>/en/todo/" | head -n 20
  # expect: HTTP/… 200
  ```

  Local preview: substitute preview origin (e.g. `http://127.0.0.1:8788`). No unwanted redirect **loop**; one 307 hop max from bare path to `/…/`.

- [ ] Page HTML redirects use **307** (document/observe), not 308.
- [ ] `Seo` canonical + hreflang for todo remain trailing-slash URLs (`createLocalizedUrl` → `…/en/todo/`).
- [ ] Optional: `curl -sI "https://<host>/api/health"` (or any `/api/*`) is **not** subject to HTML trailing-slash rewrite (worker-first / API path).
- [ ] Todo page has no native `<section>` for create/list blocks; uses `Section`.
- [ ] Visual layout on todo remains acceptable on mobile/desktop; create form internal spacing and list vertical rhythm spot-checked vs test page (extra `gap-2` is expected).
- [ ] After config change: create a todo then list still works (D1 `d1_bm` RYW session intact).
