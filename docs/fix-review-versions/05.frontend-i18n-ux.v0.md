# 05 — Frontend i18n & UX Correctness

**Priority:** Medium  
**Source:** [project-review.md](./project-review.md) findings #10, #12, #13; gap G6  
**Category:** Correctness (UI / i18n)

---

## Problem

| ID | Issue |
|----|--------|
| #10 | Delete `AlertDialogAction` closes dialog before async delete finishes; parent clears state |
| #12 | `pickLocale` returns `undefined` when a leaf lacks the active locale key |
| #13 | 404 (and other pages without i18n props) still mount Layout + i18next with undefined resources |
| G6 | `_document` `Html lang` fixed at first paint; client locale switch does not update `documentElement.lang` |

---

## Goals

- Delete confirm stays open (or only closes after success); pending state is visible; no unmount mid-flight.
- Incomplete merged locale leaves fall back to default locale string.
- Error / non-i18n pages do not break nav translations.
- `lang` attribute tracks the active locale after client-side switches.

## Implementation steps

### 1. Delete dialog race (required)

On `AlertDialogAction`:

```tsx
onClick={(e) => {
  e.preventDefault(); // stop Radix default close
  void handleConfirm();
}}
```

In `handleConfirm`:

- await `onConfirm()`
- on success: call `onOpenChange(false)` (parent clears `deletingTodo`)
- on failure: leave open; parent already toasts

Ensure parent does not clear `deletingTodo` solely from Action’s auto-close.

### 2. `pickLocale` fallback (required)

```ts
if (isLeaf) {
  const map = node as Record<string, string>;
  return (map[locale] ?? map[DEFAULT_LOCALE] ?? Object.values(map)[0]) as unknown as Messages;
}
```

### 3. `_app` / 404 i18n (required)

Pick a strategy:

| Strategy | Behavior |
|----------|----------|
| A | If `!pageProps.locale`, init with `DEFAULT_LOCALE` + empty/minimal resources; still safe `t()` |
| B | Skip `Layout` when i18n props missing (404 stays minimal shell like language landing) |
| C | Add lightweight `getStaticProps` / shared messages for 404 |

Recommended: **A + B** — 404 without full Layout chrome, and defensive i18n defaults so nothing throws.

### 4. Document `lang` (required)

On locale change (LanguageSwitcher after successful navigation, and initial mount when locale known):

```ts
document.documentElement.lang = nextLocale;
```

SSR `_document` can keep URL-based or default `lang` for first paint.

## Acceptance criteria

- [ ] Delete failure: dialog remains open; user can retry or cancel.
- [ ] Delete success: dialog closes; list refreshes.
- [ ] Missing `zh-CN` leaf with only `en` shows English (or default), not blank/`undefined`.
- [ ] Visit unknown path: no broken nav keys / no i18n crash; UX acceptable.
- [ ] Switch language in app: `document.documentElement.lang` matches new locale.
