## Goals

- Delete confirm stays open until success (or cancel); pending is visible; no unmount mid-flight.
- Incomplete merged locale leaves fall back along a defined chain; no `undefined` values in the messages tree.
- Error / non-i18n pages do not mount full Layout chrome or crash i18n init.
- `html[lang]` tracks the active `AppLocale` after client-side switches and on mount.
- Minimal a11y for delete pending: `aria-busy` on confirm; focus remains in dialog on failure.

## Implementation steps

### 1. Delete dialog race (required) — close ownership **B**

**Mechanism (R05-001):**  
`AlertDialogAction` → shadcn `AlertDialogPrimitive.Action` → Radix Dialog **Close**. Close wires `onClick` with `composeEventHandlers(props.onClick, () => context.onOpenChange(false))` and `checkForDefaultPrevented: true`. Calling `e.preventDefault()` in the user `onClick` **skips** the internal close. If the AlertDialog / Close primitive is swapped later, re-verify this path.

**Division of labor:**

| Concern | How |
|---------|-----|
| Block auto-close on confirm click | `e.preventDefault()` in Action `onClick` |
| Block double-submit | `disabled={pending}` on Action and Cancel |
| Close after success only | dialog calls `onOpenChange(false)` only after `await onConfirm()` resolves |
| Block dismiss while in-flight | dialog wrapper: `if (!nextOpen && pending) return` before forwarding `onOpenChange` |

**`delete-dialog.tsx` (B):**

```tsx
onClick={(e) => {
  e.preventDefault(); // Radix Close: composeEventHandlers honors defaultPrevented
  void handleConfirm();
}}

async function handleConfirm() {
  setPending(true);
  try {
    await onConfirm();
    onOpenChange(false); // success only
  } catch {
    // leave open; parent already toasts
  } finally {
    setPending(false);
  }
}

// controlled root:
<AlertDialog
  open
  onOpenChange={(next) => {
    if (!next && pending) return; // ignore dismiss while pending
    onOpenChange(next);
  }}
>
  ...
  <AlertDialogAction
    onClick={...}
    disabled={pending}
    aria-busy={pending}
  >
    {pending && <Spinner className="size-4" />}
    {t("todo:deleteDialog.confirm")}
  </AlertDialogAction>
```

**`todo.tsx` parent:**

```tsx
async function handleDelete() {
  if (!deletingTodo) return;
  try {
    await deleteTodo(deletingTodo.id);
    // B: do NOT setDeletingTodo(null) here
    await refresh();
    toast.success(t("todo:toast.deleted"));
  } catch (e) {
    toast.error(...);
    throw e; // optional: rethrow so dialog catch keeps open; if onConfirm swallows, dialog must not call onOpenChange(false)
  }
}

<DeleteDialog
  onConfirm={handleDelete}
  onOpenChange={(open) => {
    if (!open) setDeletingTodo(null); // sole clear / unmount entry
  }}
/>
```

**State updates (acceptance mapping):**

| Outcome | Who updates what |
|---------|------------------|
| Success close | dialog → `onOpenChange(false)` → parent `setDeletingTodo(null)`; list via `refresh` |
| Failure stay open | neither calls clear; `pending` false; dialog DOM remains |
| Cancel (not pending) | Cancel / Esc → `onOpenChange(false)` → parent clear |
| Pending dismiss attempt | ignored by dialog wrapper |

**Note on `onConfirm` errors:** Prefer `handleDelete` rethrows (or returns a rejected promise) so dialog only closes when the promise fulfills. Current parent swallows errors — **change parent to rethrow after toast**, or have `onConfirm` return `boolean` success. Recommended: rethrow after toast so B stays simple.

### 2. `pickLocale` fallback (required)

**Leaf chain (R05-003):**

```ts
if (isLeaf) {
  const map = node as Record<string, string>;
  if (typeof map[locale] === "string") return map[locale] as unknown as Messages;
  if (typeof map[DEFAULT_LOCALE] === "string")
    return map[DEFAULT_LOCALE] as unknown as Messages;
  for (const l of SUPPORTED_LOCALES) {
    if (typeof map[l] === "string") return map[l] as unknown as Messages;
  }
  return undefined as unknown as Messages; // caller omits
}
```

**Nested — omit undefined:**

```ts
const result: Messages = {};
for (const [k, v] of entries) {
  const picked = pickLocale(v, locale);
  if (picked !== undefined) result[k] = picked;
}
return result;
```

**Boundary with i18next:**

| Layer | Responsibility |
|-------|----------------|
| `pickLocale` leaf fallback | Key exists in merged JSON but active locale string missing → still emit a string |
| `fallbackMessages` + `fallbackLng` | Key missing entirely from active resource pack → i18next falls back to default locale pack |

Keep both. Do not expand `isLeaf` / fake-leaf handling in this step.

### 3. `_app` / 404 i18n (required) — strategy **A + B**

**Skip Layout (B):** same chrome policy as language landing.

```ts
const pathname = router.pathname;
const skipLayout = pathname === "/" || pathname === "/404";
```

- 404: **no** Sidebar, **no** Tabbar, **no** Toaster.
- Still wrap with `I18nextProvider` (defensive; 404 may not call `t()`).

**Defensive i18n init (A):**

```ts

  pageProps.locale && isSupportedLocale(pageProps.locale)
    ? pageProps.locale
    : DEFAULT_LOCALE;
const messages = pageProps.messages ?? {};
const fallbackMessages = pageProps.fallbackMessages ?? {};

instance.init({
  lng: locale,
  resources: {
    [locale]: messages,
    [DEFAULT_LOCALE]: fallbackMessages,
  },
  fallbackLng: DEFAULT_LOCALE,
  // ...
});
```

**404 minimal shell (`404.tsx`):** keep self-managed `messages` map; pad safe area under `viewport-fit=cover`:

```tsx
<main className="pt-[env(safe-area-inset-top)] pb-[env(safe-area-inset-bottom)] pl-[env(safe-area-inset-left)] pr-[env(safe-area-inset-right)]">
  ...
</main>
```

Unknown routes and `notFound: true` both use this same `404.tsx`.

### 4. Document `lang` (required)

- **Value:** `AppLocale` as-is (`en` | `zh-CN`) for `html[lang]` — no `zh-Hans` mapping.
- **SSR:** `_document` continues URL/`query.locale` or `DEFAULT_LOCALE` for first paint.
- **Client:** single place in `_app`:

```ts
useEffect(() => {
  const fromProps = pageProps.locale;
  const fromQuery = router.query.locale;
  const fromPath = router.pathname === "/404"
    ? router.asPath.split("/").filter(Boolean)[0]
    : undefined;
  const next =
    [fromProps, fromQuery, fromPath].find(isSupportedLocale) ?? DEFAULT_LOCALE;
  document.documentElement.lang = next;
}, [pageProps.locale, router.query.locale, router.pathname, router.asPath]);
```

LanguageSwitcher only navigates; does **not** set `lang` itself.

## Acceptance criteria

- [ ] Delete failure: dialog remains open; user can retry or cancel; focus stays inside dialog.
- [ ] Delete success: dialog closes via dialog `onOpenChange(false)` → parent clear; list refreshes.
- [ ] Confirm click: during `await`, dialog DOM stays mounted (`open`); no mid-flight unmount.
- [ ] Confirm button: `disabled={pending}` and `aria-busy={pending}`; visible label retained with Spinner.
- [ ] Pending: Cancel disabled; programmatic / Esc-driven `onOpenChange(false)` ignored.
- [ ] Missing `zh-CN` leaf with only `en`: UI shows English (or next supported string), not blank/`undefined`.
- [ ] Nested path e.g. only `en` for `todo:deleteDialog.title`: still shows fallback string; tree has no `undefined` values.
- [ ] Visit `/does-not-exist` (and `notFound`): no Layout/Tabbar; no i18n init crash / console i18n exception; text + home link tappable under notch / Home Indicator (safe-area).
- [ ] Switch en ↔ zh-CN: `document.documentElement.lang` matches UI locale.
- [ ] Hard refresh `/zh-CN/todo`: SSR `html[lang]="zh-CN"`.
